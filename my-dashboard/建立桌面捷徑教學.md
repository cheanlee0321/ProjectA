# 建立 Windows 桌面捷徑 (.lnk) 教學紀錄

> 紀錄時間：2026-05-26
> 情境：因為 Windows 對於捷徑 (.lnk) 包含中文檔名時，直接透過終端機腳本建立容易產生亂碼或錯誤。為了解決這個問題，並達成「一鍵同時啟動 Next.js 伺服器與開啟網頁」的功能，我們採用了「Node.js 產生含 BOM 編碼的 PowerShell 腳本」的完美解法。

## 為什麼要這樣做？
1. **避免中文亂碼**：直接在 CMD 執行 PowerShell 腳本時，中文檔名（如 `啟動股市儀表板.lnk`）會因為編碼衝突而失敗。透過 Node.js 寫入帶有 `\uFEFF` (UTF-8 BOM) 的檔案，能確保 PowerShell 正確解析中文。
2. **免除 `.bat` 依賴**：不需要額外的 `.bat` 執行檔，將所有指令直接寫死在 `.lnk` 捷徑的 `Arguments` 屬性中，更加純粹。
3. **自動開網頁**：利用 `cmd.exe /k` 執行複合指令，先呼叫 `start http://localhost:3000` 再執行 `npm run dev`。

---

## 產生步驟與腳本

如果未來資料夾路徑有變動，只要修改以下 Node.js 腳本中的路徑，然後執行即可重新產生捷徑。

### 步驟 1：建立 Node.js 腳本 (`create_shortcut.js`)
在專案目錄下建立一個名為 `create_shortcut.js` 的檔案，內容如下：

```javascript
const fs = require('fs');

// 注意：字串最前面的 \uFEFF 是 UTF-8 BOM，這是確保 PowerShell 看得懂中文的關鍵！
const psScript = '\uFEFF' + `
$wshell = New-Object -ComObject WScript.Shell
$desktop = [Environment]::GetFolderPath('Desktop')
$shortcutPath = Join-Path $desktop "啟動股市儀表板.lnk"
$shortcut = $wshell.CreateShortcut($shortcutPath)

# 將捷徑目標指向 cmd.exe
$shortcut.TargetPath = "cmd.exe"

# 參數：/k 代表執行完後視窗保留不關閉
# 先切換目錄 (cd /d)，然後啟動網頁 (start http...)，最後啟動伺服器 (npm run dev)
$shortcut.Arguments = '/k "cd /d C:\\Users\\chean\\OneDrive\\Desktop\\Antigravity\\ProjectA\\my-dashboard && start http://localhost:3000 && npm run dev"'
$shortcut.WorkingDirectory = "C:\\Users\\chean\\OneDrive\\Desktop\\Antigravity\\ProjectA\\my-dashboard"
$shortcut.Save()
`;

// 將這段 PowerShell 腳本輸出成 .ps1 檔案
fs.writeFileSync('C:\\Users\\chean\\OneDrive\\Desktop\\make_shortcut.ps1', psScript, 'utf8');
```

### 步驟 2：執行腳本
打開終端機 (命令提示字元或 PowerShell)，依序執行以下三個指令：

1. **讓 Node.js 產出 PowerShell 腳本**：
   ```cmd
   node create_shortcut.js
   ```

2. **執行該 PowerShell 腳本，正式在桌面產生捷徑**：
   ```cmd
   powershell -ExecutionPolicy Bypass -File C:\Users\chean\OneDrive\Desktop\make_shortcut.ps1
   ```

3. **清理暫存檔 (打掃戰場)**：
   ```cmd
   del create_shortcut.js
   del C:\Users\chean\OneDrive\Desktop\make_shortcut.ps1
   ```

大功告成！桌面上就會出現一個名為 `啟動股市儀表板.lnk` 的捷徑，雙擊它即可全自動運作！
